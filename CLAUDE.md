# 新番表工程 —— 项目知识库

## 项目概述

这是一个日本番剧信息搜集整理项目，用于搜集、整理日本番剧信息，并将结果保存为 Markdown 文件。

> **收录范围说明**：本项目仅收录 TV 动画及网络配信动画：
> - **不收录**仅在日本影院上映的剧场版动画；
> - **不收录**日本以外国家制作播放后重新日语配音播放的 TV 动画；
> - **不收录**「僧侣档」动画。

### 目录命名规则

output 目录按照「年份/月份」二级目录组织：

| 月份 | 对应季节 | 日本番剧档期 |
|:----:|:--------:|:------------:|
| 1 | 冬季 | 冬アニメ |
| 4 | 春季 | 春アニメ |
| 7 | 夏季 | 夏アニメ |
| 10 | 秋季 | 秋アニメ |

### 档期范围定义

每个档期收录番剧的开播时间范围如下：

| 档期目录 | 对应季节 | 开播时间范围 |
|:--------:|:--------:|:------------:|
| 1 | 冬季 | 12月初 ~ 2月底 |
| 4 | 春季 | 3月初 ~ 5月底 |
| 7 | 夏季 | 6月初 ~ 8月底 |
| 10 | 秋季 | 9月初 ~ 11月底 |

#### 档期判定规则

1. **以开播时间为准**：番剧归入哪个档期，由其首播日期决定，与官方宣传所标注的「春アニメ／夏アニメ」等可能不完全一致。
2. **未知具体开播时间**：仅公布所属季度（如「2026年春季」）的番剧，先归入对应档期作为候选；后期明确具体开播时间后，再根据日期重新归档。
3. **跨档期迁移**：例如某番剧最初公布为「春季档期」，后续明确 6 月开播，则需从春季档期（`4/`）剔除，迁移到夏季档期（`7/`）。
4. **索引文件同步更新**：番剧迁移时，需同步更新两个档期的 `_index.md` 文件。

### 月份目录内容

每个月份目录下包含：

1. **番剧信息文件**（每部番剧一个 Markdown 文件）
   - 命名规则：优先使用**中文译名**；无译名则使用**日文原名**
   - **禁止字符处理**：若译名或原名中含有 Windows/Linux 禁止用作文件名的字符（`\ / : * ? " < > |` 等），将其替换为下划线 `_`
   - 示例：
     - 有译名：`雾尾粉丝后援会.md`
     - 无译名：`霧尾ファンクラブ.md`
     - 含禁止字符：`无职转生：到了异世界_.md`（原译名中的 `:` 替换为 `_`）

2. **番剧名称索引文件**（固定文件名：`_index.md`）
   - 记录该季度所有番剧的名称对应关系

### 网络代理配置

当使用 `curl` 等命令行工具检索网络信息时，若遇到站点被防火墙拦截的情况，使用以下代理：

- **HTTP 代理**：`http://192.168.2.200:2080`
- **HTTPS 代理**：`http://192.168.2.200:2080`

在 `curl` 中使用时添加 `--proxy http://192.168.2.200:2080` 参数。

#### 强制代理规则

以下情况**一律使用代理**，无需判断是否被拦截：

- 番剧官方网站
- `.jp` 结尾域名的网站
- 中国境外网站（如 X / 推特、Facebook 等）

### 网页读取工具优先级

读取网页时，工具使用优先级如下：

1. **Web Fetch**（优先使用）
2. **`curl` 等命令行工具**（Web Fetch 失败时 fallback）

> 使用 `curl` 时若遇到被防火墙拦截的站点，请参考上方「网络代理配置」添加代理参数。

### 自定义 STAFF 译名

文件 `data/staff-name-trans.md` 用于自定义 STAFF 人员的中文译名。

#### 使用方式

在搜集 STAFF 信息时，姓名优先级如下：
1. Bangumi 译名
2. 其他来源译名
3. **staff-name-trans.md 中的自定义译名**
4. 原名（日文名或英文名）

#### 文件格式示例

```markdown

新海誠 -> 新海诚
```

### 自定义 CAST 译名

文件 `data/cast-name-trans.md` 用于自定义声优（CAST）姓名的中文译名。

#### 使用方式

在搜集 CAST 信息时，**声优名**翻译优先级如下：
1. **Bangumi 译名**
2. **其他来源译名**（萌娘百科、百度百科、官方网站等）
3. **cast-name-trans.md 中的自定义译名**
4. **原名**（日文名或英文名）

#### 角色名翻译规则

**角色名**翻译优先级如下：
1. **Bangumi 译名**
2. **萌娘百科等来源译名**
3. **无参考译名则自行翻译**（遵循中文翻译习惯，保持与作品风格一致）
4. **原名**（实在无法翻译时）

#### 文件格式示例

```markdown
# CAST 姓名中文译名参考

花澤香菜 -> 花泽香菜

神谷浩史 -> 神谷浩史
```

每部番剧需要搜集以下信息：

| 字段 | 说明 | 数据来源 |
|------|------|----------|

| **英文译名** | 番剧的英文译名 | anime-planet |

| **故事简介** | 剧情简介（**必须翻译为简体中文**） | Bangumi、萌娘百科 |
| **主要 CAST** | 主要声优/配音演员（角色名、声优名翻译规则见下文） | Bangumi、萌娘百科、官方网站、cast-name-trans.md |
| **主要 STAFF** | 主要制作人员（见下方 STAFF 记录规则） | Bangumi、官方网站、staff-name-trans.md |
| **官网网址** | 番剧官方网站 | Bangumi、anime-planet |
| **开播时间** | 首播日期（格式规则见下文） | Bangumi |

| **TAG** | 描述番剧剧情的简短标签（多个，见下方规则） | Bangumi、anime-planet、剧情检索 |

### 全球独占判定规则

#### 判定条件（需同时满足）

1. 番剧有**公开开播时间**
2. 该季度播放档期内**没有日本电视台播放计划**
3. **只有一家**线上流媒体平台播放

#### 标记规则

| 情况 | 标记值 |
|------|--------|
| 不满足全球独占条件 | `-` |
| 满足全球独占条件 | 平台名称，如 `Netflix`、`Disney+`、`Amazon Prime Video` 等 |

#### 示例

| 番剧 | 电视台 | 线上平台 | 标记 |
|------|--------|----------|------|
| A | 东京电视台 | Netflix | `-` |
| B | 无 | Netflix、Crunchyroll | `-` |
| C | 无 | Netflix | `Netflix` |
| D | 无 | Disney+ | `Disney+` |

### 泡面番判定规则

#### 判定方式

1. **优先从 Bangumi 的 TAG 中判断**（如包含 `泡面` 相关标签）
2. 若 Bangumi TAG 无法明确判断，可参考其他数据源的分类标签、集数时长、作品介绍等信息综合判断
3. 单集时长通常在 **8 分钟以内** 的短篇动画，可判定为泡面番

#### 标记规则

| 情况 | 标记值 |
|------|--------|
| 不是泡面番 | `-` |
| 是泡面番 | `是` |

### 多版本判定规则

#### 判定方式

若动画在**不同平台区分不同版本播放**（例如修正版、无修正版、TV版、完整版等），则判定为存在多版本。

#### 数据来源优先级

1. **动画官网的播放信息（ON AIR）**
2. 其他信息来源（Bangumi、各数据源等）

#### 标记规则

| 情况 | 标记值 |
|------|--------|
| 无多版本 | `-` |
| 存在多版本 | `是` |

### TAG 字段规则

#### 定义

TAG 字段用于以**简短词语**描述番剧的核心剧情特征与题材属性，便于快速了解作品定位与风格。

#### 搜集原则

1. **参考数据源 TAG**：优先参考 Bangumi 和 anime-planet 上的用户/官方标签。
2. **剧情校验**：必须结合故事简介、PV、官方网站等**多方剧情信息**进行校验，剔除与剧情不符的标签，补充遗漏的关键标签。
3. **归纳精简**：最终输出的 TAG 应为**精简后的中文词语**，避免过长或过于冷门的表达。

#### 输出格式

- 多个 TAG 之间用中文顿号「、」分隔
- 每个 TAG 控制在 2~6 个汉字，特殊情况（如英文专有名词）除外
- 如无合适 TAG，标记为 `-`
- TAG 数量建议控制在 **3~8 个**

#### 常见 TAG 类别参考

| 类别 | 示例 |
|------|------|
| 题材类型 | 校园、职场、运动、音乐、美食、旅行、战争、推理、悬疑 |
| 核心元素 | 魔法、异世界、转生、时间穿越、机甲、吸血鬼、僵尸、恋爱、战斗 |
| 情感基调 | 治愈、催泪、搞笑、热血、致郁、恐怖、日常、励志 |
| 受众指向 | 乙女向、后宫、百合、基腐、子供向、女性向 |
| 风格特征 | 赛博朋克、废土、蒸汽朋克、萌系、写实、意识流 |

#### 注意事项

- TAG 应聚焦于**剧情内容本身**，而非制作信息（如「新房昭之」「京都动画」不应作为 TAG）
- 避免过于笼统的标签（如「动画」「日本」）
- 续作番剧需结合**当季剧情**归纳 TAG，而非沿用全系列通用标签

### 开播时间格式规则

根据信息公开程度，采用以下格式：

| 信息完整度 | 格式 | 示例 |
|------------|------|------|
| 公开具体日期和时间 | `M月D日 HH:MM` | `4月1日 00:05` |
| 公开日期，未公开时间 | `M月D日` | `4月1日` |
| 未公开具体日期 | `YYYY年X季` | `2026年春季` |

**注意**：默认隐去年份（未公开日期的情况除外）。

#### 时区规则（必须）

「开播时间」**一律记录为北京时间（UTC+8）**。搜集番剧开播时间时必须先确认数据源的时区，再转换为北京时间后写入文件。

##### 各数据源默认时区

| 数据源类型 | 默认时区 | 与北京时间的差 |
|------------|----------|:--------------:|
| 日文资料、日文官网、日本电视台官网、Bangumi（多数情况） | 日本时间（JST，UTC+9） | 北京时间 = 日本时间 − 1 小时 |
| 中文资料（B 站、微博、萌娘百科等） | 一般已是北京时间 | 不需换算 |
| 其他时区（流媒体平台等） | 视具体平台而定 | 按各自时差换算 |

> **判断原则**：日文页面、`.jp` 域名、官方电视台时刻表等**默认按 JST 处理**；如果数据源明确标注了时区或已转换为其他时区，则以标注为准。

##### 日本「跨夜计时法」（25:00、26:00 等）

日本深夜动画常使用「跨夜计时法」，将 24:00 ~ 29:59 表示为前一日的延续（如「25:30」表示「次日 1:30」）。**记录前必须先转换为标准 24 小时制（00:00 ~ 23:59），再换算到北京时间**。

##### 转换示例

| 数据源（日本时间） | 等价标准 JST | 北京时间（最终记录值） |
|--------------------|--------------|------------------------|
| `1月2日 23:30` | 1月2日 23:30 | `1月2日 22:30` |
| `1月2日 25:30` | 1月3日 01:30 | `1月3日 00:30` |
| `7月5日 24:00` | 7月6日 00:00 | `7月5日 23:00` |
| `7月5日 26:30` | 7月6日 02:30 | `7月6日 01:30` |

> **关键提醒**：跨夜计时法的「日期」是按播出当晚的归属编排的（如周六深夜的 25:00 实际是周日凌晨），转换时必须**同时调整日期与时间**，不要直接把 25:30 写成 25:30 或仅减 1 小时得到 24:30。

### 索引表格中的文件名须用反引号包裹

索引文件 `_index.md` 的表格中，**「文件名」列的值一律用反引号 `` ` `` 包裹**，防止文件名中的 `!`、`[]`、`()`、`~` 等符号被 Markdown 解析器误识别为格式标记。

| 正确写法 | 错误写法 |
|:--------:|:--------:|
| `` `雾尾粉丝后援会.md` `` | `雾尾粉丝后援会.md` |
| `` `无职转生：到了异世界_.md` `` | `无职转生：到了异世界_.md` |

> **原则**：文件名本身的内容不变（含已被替换为下划线的禁止字符），仅在索引表格中额外添加反引号作为 Markdown 行内代码保护。

### Markdown 控制字符须转义或包裹

在番剧名称（译名、别名、日文原名、英文名）以及 STAFF / CAST 的人名、角色名中，若出现会被 Markdown 解析器误识别的特殊符号，**必须做转义处理**，以保证渲染结果与原始名称一致。

#### 须处理的字符

| 字符 | Markdown 语义 | 处理方式 |
|:----:|---------------|----------|
| `\|` | 表格列分隔符（仅表格环境） | 用反引号将整段内容包裹为行内代码 |
| `` ` `` | 行内代码标记 | 用反引号将整段内容包裹，或改用两个反引号包围（如 ``` `` ` `` ```） |
| `*` | 粗体 / 斜体 | 前面加反斜杠 `\*` 或用反引号包裹 |
| `_` | 粗体 / 斜体 | 前面加反斜杠 `\_` 或用反引号包裹 |
| `~` | 删除线 | 前面加反斜杠 `\~` 或用反引号包裹 |
| `[` / `]` | 链接 / 图片 | 前面加反斜杠 `\[` `\]` 或用反引号包裹 |
| `#` | 标题（若出现在行首） | 前面加反斜杠 `\#` 或用反引号包裹 |

#### 处理优先级

1. **优先使用反引号包裹**：将含特殊符号的整段名称包裹为行内代码（`` `名称` ``），最为稳妥且不易遗漏
2. **次选反斜杠转义**：在特殊字符前加 `\`，适用于仅含单个特殊字符且不宜改变视觉样式的场景

#### 应用范围

| 输出位置 | 示例 |
|----------|------|
| `_index.md` 表格各列 | `` `无职转生：到了异世界_.md` `` |
| 番剧信息文件的标题 `# {译名}` | `# 无职转生\_\_到了异世界`（若译名含 `_`） |
| 番剧信息文件的列表字段 | `- 英文名：\`Mushoku Tensei\_\_Jobless Reincarnation\`` |
| STAFF / CAST 列表 | `- 导演：\`友田\_康\`` |

> **原则**：转义仅是为了让 Markdown 正确渲染，**不得因此改动原始名称的字符本身**。例如原名含 `_`，转义后仍是 `_`，只是写作 `\_` 或将其放在反引号内。

### 中文标点符号（适用于所有文件）

**所有文件**（包括 `CLAUDE.md`、`_index.md`、番剧信息文件等项目内任何 Markdown / 文本文件）中，中文语境下的标点必须使用**全角中文标点**，不得使用半角英文标点。**最常见的失误是把逗号写成英文「,」、把冒号写成英文「:」**，需特别注意。

| 用途 | 应使用（全角） | 不可使用（半角） |
|:----:|:--------------:|:----------------:|
| 句中停顿 | `，` | `,` |
| 句末 | `。` | `.` |
| 列举 | `、` | （无对应） |
| 分号 | `；` | `;` |
| 冒号 | `：` | `:` |
| 问号 | `？` | `?` |
| 感叹号 | `！` | `!` |
| 引号 | `「」` 或 `“”` | `""` |
| 括号 | `（` `）` | `(` `)` |
| 破折号 | `——` | `--` 或 `—` |
| 省略号 | `……` | `...` |

**例外**：中文文本中如果包含**英文专有名词、英文句子、数字、URL、时间（如 `00:05`）、代码（如 inline `code`）、Markdown 表格分隔符（`|:---:|`）** 等英文/技术上下文，其内部固有的英文标点保持原样。

> **原则**：中文写中文标点，英文写英文标点，不跨语境混用。
