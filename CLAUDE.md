# 新番表工程 —— 项目知识库

## 项目概述

这是一个日本番剧信息搜集整理项目，用于搜集、整理日本番剧信息，并将结果保存为 Markdown 文件。

> **收录范围说明**：本项目仅收录 TV 动画及网络配信动画：
> - **不收录**仅在日本影院上映的剧场版动画；
> - **不收录**日本以外国家制作播放后重新日语配音播放的 TV 动画；
> - **不收录**「僧侣档」动画；
> - **不收录**R18 / 里番，除非用户明确指示整理某部特定作品。

### 目录命名规则

output 目录按照「年份/月份」二级目录组织：

| 月份 | 对应季节 | 日本番剧档期 |
|:----:|:--------:|:------------:|
| 1 | 冬季 | 冬アニメ |
| 4 | 春季 | 春アニメ |
| 7 | 夏季 | 夏アニメ |
| 10 | 秋季 | 秋アニメ |

### 档期范围定义

每个档期收录番剧的开播时间范围如下：

| 档期目录 | 对应季节 | 开播时间范围 |
|:--------:|:--------:|:------------:|
| 1 | 冬季 | 12月初 ~ 2月底 |
| 4 | 春季 | 3月初 ~ 5月底 |
| 7 | 夏季 | 6月初 ~ 8月底 |
| 10 | 秋季 | 9月初 ~ 11月底 |

#### 档期判定规则

1. **以开播时间为准**：番剧归入哪个档期，由其首播日期决定，与官方宣传所标注的「春アニメ／夏アニメ」等可能不完全一致。
2. **未知具体开播时间**：仅公布所属季度（如「2026年春季」）的番剧，先归入对应档期作为候选；后期明确具体开播时间后，再根据日期重新归档。
3. **跨档期迁移**：例如某番剧最初公布为「春季档期」，后续明确 6 月开播，则需从春季档期（`4/`）剔除，迁移到夏季档期（`7/`）。**执行跨档期迁移前须询问用户确认**。
4. **索引文件同步更新**：番剧迁移时，需同步更新两个档期的 `_index.md` 文件。
5. **删除确认**：若番剧因取消播放计划或制作计划等原因需要从项目中移除（包括删除番剧信息文件及从索引中剔除），**必须询问用户确认**后再执行。

### 月份目录内容

每个月份目录下包含：

1. **番剧信息文件**（每部番剧一个 Markdown 文件）
   - 命名规则：优先使用**中文译名**；无译名则使用**日文原名**
   - **文件名符号处理**：文件名中若出现**任何符号**（包括空格、全角/半角符号如 `!`、`?`、`~`、`()`、`[]`、`:` 等），均须替换为下划线 `_`；**连续多个符号**须合并替换为**单个**下划线 `_`
   - 示例：
     - 有译名：`雾尾粉丝后援会.md`
     - 无译名：`霧尾ファンクラブ.md`
     - 含符号：`无职转生_到了异世界_.md`（原译名中的 `：` 替换为 `_`）

2. **番剧名称索引文件**（固定文件名：`_index.md`）
   - 记录该季度所有番剧的名称对应关系

### 完整目录示例

```
output/
└── 2026/
    └── 4/                      # 2026年春季番剧
        ├── _index.md           # 番剧名称索引

        ├── 轮回的花瓣.md
```

## 番剧名称索引规范

索引文件 `_index.md` 记录该季度所有番剧的名称信息。

### 字段说明

| 字段 | 说明 | 数据来源 | 示例 |
|------|------|----------|------|
| **译名** | 优先采用的中文译名 | Bangumi | `雾尾粉丝后援会` |
| **日文原名** | 番剧的日文原名 | Bangumi | `霧尾ファンクラブ` |
| **别名** | 其他中文译名（可能有多个） | 各数据源 | `雾尾粉丝俱乐部` |
| **英文名** | 番剧的英文译名 | anime-planet | `Kirio Fan Club` |
| **文件名** | 番剧信息文件的文件名（含 `.md` 后缀），与「译名」一致，但文件名中**任何符号**（含空格）均替换为下划线 `_`，**连续多个符号合并为单个**下划线 `_`；在索引表格中须用反引号包裹以避免 Markdown 误解析 | 由译名推导 | `` `雾尾粉丝后援会.md` `` |

### 索引文件格式示例

```markdown
# 2026年春季番剧索引

| 译名 | 日文原名 | 别名 | 英文名 | 文件名 |
|------|----------|------|--------|--------|
| 雾尾粉丝后援会 | 霧尾ファンクラブ | 雾尾粉丝俱乐部 | Kirio Fan Club | `雾尾粉丝后援会.md` |
| 轮回的花瓣 | リィンカーネーションの花弁 | - | Petals of Reincarnation | `轮回的花瓣.md` |
```

### 索引表格中的文件名须用反引号包裹

索引文件 `_index.md` 的表格中，**「文件名」列的值一律用反引号 `` ` `` 包裹**，防止文件名中的 `!`、`[]`、`()`、`~` 等符号被 Markdown 解析器误识别为格式标记。

| 正确写法 | 错误写法 |
|:--------:|:--------:|
| `` `雾尾粉丝后援会.md` `` | `雾尾粉丝后援会.md` |
| `` `无职转生：到了异世界_.md` `` | `无职转生：到了异世界_.md` |

> **原则**：文件名本身的内容不变（含已被替换为下划线的禁止字符），仅在索引表格中额外添加反引号作为 Markdown 行内代码保护。

## Markdown 控制字符须转义或包裹

在番剧名称（译名、别名、日文原名、英文名）以及 STAFF / CAST 的人名、角色名中，若出现会被 Markdown 解析器误识别的特殊符号，**必须做转义处理**，以保证渲染结果与原始名称一致。

### 须处理的字符

| 字符 | Markdown 语义 | 处理方式 |
|:----:|---------------|----------|
| `\|` | 表格列分隔符（仅表格环境） | 用反引号将整段内容包裹为行内代码 |
| `` ` `` | 行内代码标记 | 用反引号将整段内容包裹，或改用两个反引号包围（如 ``` `` ` `` ```） |
| `*` | 粗体 / 斜体 | 前面加反斜杠 `\*` 或用反引号包裹 |
| `_` | 粗体 / 斜体 | 前面加反斜杠 `\_` 或用反引号包裹 |
| `~` | 删除线 | 前面加反斜杠 `\~` 或用反引号包裹 |
| `[` / `]` | 链接 / 图片 | 前面加反斜杠 `\[` `\]` 或用反引号包裹 |
| `#` | 标题（若出现在行首） | 前面加反斜杠 `\#` 或用反引号包裹 |

### 处理优先级

1. **优先使用反引号包裹**：将含特殊符号的整段名称包裹为行内代码（`` `名称` ``），最为稳妥且不易遗漏
2. **次选反斜杠转义**：在特殊字符前加 `\`，适用于仅含单个特殊字符且不宜改变视觉样式的场景

### 应用范围

| 输出位置 | 示例 |
|----------|------|
| `_index.md` 表格各列 | `` `无职转生：到了异世界_.md` `` |
| 番剧信息文件的标题 `# {译名}` | `# 无职转生\_\_到了异世界`（若译名含 `_`） |
| 番剧信息文件的列表字段 | `- 英文名：\`Mushoku Tensei\_\_Jobless Reincarnation\`` |
| STAFF / CAST 列表 | `- 导演：\`友田\_康\`` |

> **原则**：转义仅是为了让 Markdown 正确渲染，**不得因此改动原始名称的字符本身**。例如原名含 `_`，转义后仍是 `_`，只是写作 `\_` 或将其放在反引号内。

## 中文标点符号（适用于所有文件）

**所有文件**（包括 `CLAUDE.md`、`_index.md`、番剧信息文件等项目内任何 Markdown / 文本文件）中，中文语境下的标点必须使用**全角中文标点**，不得使用半角英文标点。**最常见的失误是把逗号写成英文「,」、把冒号写成英文「:」**，需特别注意。

| 用途 | 应使用（全角） | 不可使用（半角） |
|:----:|:--------------:|:----------------:|
| 句中停顿 | `，` | `,` |
| 句末 | `。` | `.` |
| 列举 | `、` | （无对应） |
| 分号 | `；` | `;` |
| 冒号 | `：` | `:` |
| 问号 | `？` | `?` |
| 感叹号 | `！` | `!` |
| 引号 | `「」` 或 `“”` | `""` |
| 括号 | `（` `）` | `(` `)` |
| 破折号 | `——` | `--` 或 `—` |
| 省略号 | `……` | `...` |

**例外**：中文文本中如果包含**英文专有名词、英文句子、数字、URL、时间（如 `00:05`）、代码（如 inline `code`）、Markdown 表格分隔符（`|:---:|`）** 等英文/技术上下文，其内部固有的英文标点保持原样。

> **原则**：中文写中文标点，英文写英文标点，不跨语境混用。

### 网络代理配置

当使用 `curl` 等命令行工具检索网络信息时，若遇到站点被防火墙拦截的情况，使用以下代理：

- **HTTP 代理**：`http://127.0.0.1:2080`
- **HTTPS 代理**：`http://127.0.0.1:2080`

在 `curl` 中使用时添加 `--proxy http://127.0.0.1:2080` 参数。

#### 强制代理规则

以下情况**一律使用代理**，无需判断是否被拦截：

- 番剧官方网站
- `.jp` 结尾域名的网站
- 中国境外网站（如 X / 推特、Facebook 等）

### 网页读取工具优先级

读取网页时，工具使用优先级如下：

1. **Web Fetch**（优先使用；若执行失败，须重试 2 次；重试 2 次仍失败后，再使用 `curl` 等工具）
2. **`curl` 等命令行工具**（Web Fetch 重试 2 次均失败后 fallback）

> 使用 `curl` 时若遇到被防火墙拦截的站点，请参考上方「网络代理配置」添加代理参数。

### 整理规则

**默认规则**：
- 所有信息默认以 **Bangumi** 的数据为优先参考
- 番剧的**英文译名**以 **anime-planet** 为准
