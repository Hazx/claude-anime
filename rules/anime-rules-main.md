# 番剧信息整理总体规则

编写和维护番剧信息内容时，须遵循以下总体规则。

## 番剧信息字段

每部番剧需要搜集以下信息：

| 字段 | 说明 | 数据来源 |
|------|------|----------|
| **译名** | 优先采用的中文译名 | Bangumi |
| **别名** | 其他中文译名（可能有多个） | Bangumi、萌娘百科 |
| **日文原名** | 番剧的日文原名 | Bangumi |
| **英文译名** | 番剧的英文译名 | anime-planet |
| **番剧类型** | 类型标签（原创/小说/漫画/游戏改编，可带国籍前缀） | Bangumi、anime-planet |
| **故事简介** | 剧情简介（**必须翻译为简体中文**） | Bangumi、萌娘百科 |
| **主要 CAST** | 主要声优/配音演员（角色名、声优名翻译规则见下文） | Bangumi、萌娘百科、官方网站、cast-name-trans.md |
| **主要 STAFF** | 主要制作人员（见下方 STAFF 记录规则） | Bangumi、官方网站、staff-name-trans.md |
| **官网网址** | 番剧官方网站 | Bangumi、anime-planet |
| **开播时间** | 首播日期（格式规则见下文） | Bangumi |
| **集数** | 集数信息（格式规则见下文） | Bangumi、AniChart |
| **全球独占** | 是否为全球线上平台独占 | Bangumi、官方网站 |
| **泡面番** | 是否为泡面番（单集时长通常在 8 分钟以内） | Bangumi、各数据源 |
| **多版本** | 是否在不同平台区分不同版本播放 | Bangumi、各数据源 |
| **TAG** | 描述番剧剧情的简短标签（多个，见 `anime-rules-tag.md`） | Bangumi、anime-planet、剧情检索 |

### 信息维护与更新规则

在维护已有番剧信息时，若搜集到与现有内容不同的新内容，须按以下原则处理：

1. **直接更新**：新内容与老内容方向一致，但更为详尽或准确，可直接替换更新。
2. **多方甄别**：新内容与老内容差异较大，或涉及删除已有内容时，须通过多方查询确认新旧内容的出入来源，判断哪一方更可靠。
3. **询问确认**：经甄别后仍无法判断，或变动涉及删除、大幅改写等关键操作时，须询问用户确认后再执行。
4. **删除确认**：若番剧因取消播放计划或制作计划等原因需要从项目中移除，**必须询问用户确认**后再执行删除。
5. **跨季度迁移确认**：若番剧因延期等原因需要从一个季度迁移到另一个季度（如从 4 月档延期至 10 月档），**必须询问用户确认**后再执行迁移。
6. **名称联动更新**：更新番剧的译名、别名、日文原名或英文译名时，须同步检查并更新以下内容，确保各处保持一致：
   - 当季索引文件 `_index.md` 中对应的名称字段
   - 番剧内容 Markdown 文件的文件名（按「译名优先，任何符号（含空格）替换为单个下划线 `_`」规则重新命名）
   - 索引文件中记录的「文件名」须与番剧内容 Markdown 文件的实际文件名一致

### 全球独占判定规则

#### 判定条件（需同时满足）

1. 番剧有**公开开播时间**
2. 该季度播放档期内**没有日本电视台播放计划**
3. **只有一家**线上流媒体平台播放

#### 标记规则

| 情况 | 标记值 |
|------|--------|
| 不满足全球独占条件 | `-` |
| 满足全球独占条件 | 平台名称，如 `Netflix`、`Disney+`、`Amazon Prime Video` 等 |

#### 示例

| 番剧 | 电视台 | 线上平台 | 标记 |
|------|--------|----------|------|
| A | 东京电视台 | Netflix | `-` |
| B | 无 | Netflix、Crunchyroll | `-` |
| C | 无 | Netflix | `Netflix` |
| D | 无 | Disney+ | `Disney+` |

### 泡面番判定规则

#### 判定方式

1. **优先从 Bangumi 的 TAG 中判断**（如包含 `泡面` 相关标签）
2. 若 Bangumi TAG 无法明确判断，可参考其他数据源的分类标签、集数时长、作品介绍等信息综合判断
3. 单集时长通常在 **8 分钟以内** 的短篇动画，可判定为泡面番

#### 标记规则

| 情况 | 标记值 |
|------|--------|
| 不是泡面番 | `-` |
| 是泡面番 | `是` |

### 多版本判定规则

#### 判定方式

若动画在**不同平台区分不同版本播放**（例如修正版、无修正版、TV版、完整版等），则判定为存在多版本。

#### 数据来源优先级

1. **动画官网的播放信息（ON AIR）**
2. 其他信息来源（Bangumi、各数据源等）

#### 标记规则

| 情况 | 标记值 |
|------|--------|
| 无多版本 | `-` |
| 存在多版本 | `是` |

### TAG 字段规则

编写和维护番剧信息的 TAG 字段时，须遵循 `anime-rules-tag.md` 中的规则。

### 番剧类型

#### 基础类型

| 类型 | 说明 |
|------|------|
| **原创动画** | 原创剧本，无原作基础 |
| **小说改编** | 改编自轻小说、小说等文字作品 |
| **漫画改编** | 改编自漫画作品 |
| **游戏改编** | 改编自游戏作品 |
| **其他类型** | 遇到时再具体定义 |

#### 国籍标注规则

- 如果原作是**日本**作品：**不标注**国籍（默认情况）
- 如果原作是**非日本**作品：在类型前加上**国家/地区名**

#### 示例

| 情况 | 最终类型 |
|------|----------|
| 日本漫画改编 | `漫画改编` |
| 韩国漫画改编 | `韩国漫画改编` |
| 中国小说改编 | `中国小说改编` |
| 美国游戏改编 | `美国游戏改编` |
| 日本原创 | `原创动画` |

### 开播时间格式规则

根据信息公开程度，采用以下格式：

| 信息完整度 | 格式 | 示例 |
|------------|------|------|
| 公开具体日期和时间 | `M月D日 HH:MM` | `4月1日 00:05` |
| 公开日期，未公开时间 | `M月D日` | `4月1日` |
| 未公开具体日期 | `YYYY年X季` | `2026年春季` |

**注意**：默认隐去年份（未公开日期的情况除外）。

#### 时区规则（必须）

「开播时间」**一律记录为北京时间（UTC+8）**。搜集番剧开播时间时必须先确认数据源的时区，再转换为北京时间后写入文件。

##### 各数据源默认时区

| 数据源类型 | 默认时区 | 与北京时间的差 |
|------------|----------|:--------------:|
| 日文资料、日文官网、日本电视台官网、Bangumi（多数情况） | 日本时间（JST，UTC+9） | 北京时间 = 日本时间 − 1 小时 |
| 中文资料（B 站、微博、萌娘百科等） | 一般已是北京时间 | 不需换算 |
| 其他时区（流媒体平台等） | 视具体平台而定 | 按各自时差换算 |

> **判断原则**：日文页面、`.jp` 域名、官方电视台时刻表等**默认按 JST 处理**；如果数据源明确标注了时区或已转换为其他时区，则以标注为准。

##### 日本「跨夜计时法」（25:00、26:00 等）

日本深夜动画常使用「跨夜计时法」，将 24:00 ~ 29:59 表示为前一日的延续（如「25:30」表示「次日 1:30」）。**记录前必须先转换为标准 24 小时制（00:00 ~ 23:59），再换算到北京时间**。

##### 转换示例

| 数据源（日本时间） | 等价标准 JST | 北京时间（最终记录值） |
|--------------------|--------------|------------------------|
| `1月2日 23:30` | 1月2日 23:30 | `1月2日 22:30` |
| `1月2日 25:30` | 1月3日 01:30 | `1月3日 00:30` |
| `7月5日 24:00` | 7月6日 00:00 | `7月5日 23:00` |
| `7月5日 26:30` | 7月6日 02:30 | `7月6日 01:30` |

> **关键提醒**：跨夜计时法的「日期」是按播出当晚的归属编排的（如周六深夜的 25:00 实际是周日凌晨），转换时必须**同时调整日期与时间**，不要直接把 25:30 写成 25:30 或仅减 1 小时得到 24:30。

### 集数格式规则

#### 普通情况

| 情况 | 格式 | 示例 |
|------|------|------|
| 已确定集数 | `共X集` | `共12集` |

#### 分割放送（一季企划分多季度播放）

| 情况 | 格式 | 示例 |
|------|------|------|
| 公布多季度长度，分割播放，已知总计数 | `共Y集 播X集` | `共24集 播12集` |
| 公布多季度长度，分割播放，已知当季集数 | `共2季 播X集` | `共2季 播13集` |
| 公布多季度长度，分割播放，未知具体集数 | `共2季 播1季` | `共2季 播1季` |

#### 未公开具体集数

| 播放档期 | 标记 |
|----------|------|
| 一季度 | `季番` |
| 两季度 | `半年番` |
| 三季度 | `三季度` |
| 四季度 | `年番` |

### STAFF 记录规则

每部番剧的 STAFF 信息分**两部分**记录。

不同数据源对同一 STAFF 职位可能使用不同的名称（如日文原名、不同中文译法）。整理时须参考 [`staff-position-trans.md`](staff-position-trans.md) 将职位名称统一为项目标准名称后，再写入各个 STAFF 列表。

#### 第一部分：完整 STAFF 列表

记录从数据源获取到的**所有** STAFF 信息，按原样列出。

#### 第二部分：精选 STAFF 列表

从完整列表中挑选以下关键职位，按**指定顺序**记录（仅列出实际存在的职位）：

| 顺序 | 职位 |
|:----:|------|
| 1 | 总导演 |
| 2 | 导演 |
| 3 | 系列导演 |
| 4 | 副导演 |
| 5 | 编剧 |
| 6 | 系列构成 |
| 7 | 角色原案 |
| 8 | 人设 |
| 9 | 总作画导演 |
| 10 | 3D导演 |
| 11 | CG导演 |
| 12 | 音乐 |
| 13 | 动画制作人 |
| 14 | 动画制作 |
| 15 | 制片 |

> **注意**：姓名翻译优先级与整体 STAFF 规则一致：Bangumi 译名 → 其他来源译名 → staff-name-trans.md → 原名。


## 自定义 STAFF 译名

文件 `staff-name-trans.md` 用于自定义 STAFF 人员的中文译名。

### 使用方式

在搜集 STAFF 信息时，姓名优先级如下：
1. Bangumi 译名
2. 其他来源译名
3. **staff-name-trans.md 中的自定义译名**
4. 原名（日文名或英文名）

### 文件格式示例

```markdown
# STAFF 姓名中文译名参考

山田尚子 -> 山田尚子

新海誠 -> 新海诚
```

## 自定义 CAST 译名

文件 `cast-name-trans.md` 用于自定义声优（CAST）姓名的中文译名。

### 使用方式

在搜集 CAST 信息时，**声优名**翻译优先级如下：
1. **Bangumi 译名**
2. **其他来源译名**（萌娘百科、百度百科、官方网站等）
3. **cast-name-trans.md 中的自定义译名**
4. **原名**（日文名或英文名）

### 角色名翻译规则

**角色名**翻译优先级如下：
1. **Bangumi 译名**
2. **萌娘百科等来源译名**
3. **无参考译名则自行翻译**（遵循中文翻译习惯，保持与作品风格一致）
4. **原名**（实在无法翻译时）

### 文件格式示例

```markdown
# CAST 姓名中文译名参考

花澤香菜 -> 花泽香菜

神谷浩史 -> 神谷浩史
```
